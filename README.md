# ドラえもん人狼 (Doraemon Werewolf)

1台のスマートフォンを参加者間で回しながらプレイする、対面型の特殊人狼ゲームアプリケーション。
ドラえもんのキャラクターをモチーフにした多様な役職と、複雑な能力処理をブラウザ上で完全自動化している。

## 特徴

* [cite_start]**1台で遊べる（Pass & Play方式）**: 参加者全員がアプリをインストールする必要はなく、1台の端末を順に回して「能力確認」「夜の行動」を行う [cite: 8]。
* **ゲームマスター（GM）不要**: 複雑な夜の行動処理、能力の優先度判定、勝利判定などはすべてプログラムが自動で行う。
* **特殊な役職とカオスな展開**: 自分が何の役職か勘違いしている「のび太」や、ランダムに暴れまわる「神成さん」など、通常の人狼にはない予測不能な展開が楽しめる。

### ゲームの流れ

1. [cite_start]**Config（設定）**: 役職人数、プレイヤー名、会話時間、初日投票の有無などを設定する [cite: 2, 3, 4, 5]。
2. [cite_start]**能力確認**: スマホを回し、各プレイヤーが自分の役職を確認する [cite: 8]。
3. [cite_start]**会話ターン**: 設定された時間（デフォルト5分）内で議論を行う。タイマー終了後、投票へ移る [cite: 9, 11]。
4. [cite_start]**能力ターン（夜の行動）**: 生存者が順に自身の能力を行使する対象を選択する [cite: 13]。
5. [cite_start]**結果表示**: 全員の行動が確定した後、気絶者や能力の使用結果が開示される [cite: 16]。
6. [cite_start]**判定**: 各陣営の勝利条件を満たしたか判定される [cite: 17, 18, 19, 20]。

## 役職一覧

[cite_start]各プレイヤーの行動履歴は記録され、能力使用時には過去の履歴もUI上に表示される [cite: 24]。

### 🐺 人狼陣営

[cite_start]人狼の人数が村人の人数以上になれば勝利 [cite: 18][cite_start]。ジャイアンとメカジャイアンはお互いの正体を認識している [cite: 25]。

* [cite_start]**ジャイアン（人狼）**: 毎ターン1人を殴って気絶させる [cite: 27]。
* [cite_start]**メカジャイアン**: ジャイアンの襲撃能力か、現在場にいる「本物の村人」の能力のどちらかを選択して行使できる（同時使用不可） [cite: 29]。
* [cite_start]**スネ夫（狂人）**: 能力はない。勝利判定は人狼側だが、人狼からは誰がスネ夫か分からない [cite: 31, 32]。

### 🏡 村人陣営

[cite_start]人狼を全員追放すれば勝利 [cite: 19]。

* [cite_start]**ドラえもん（占い師）**: 対象の役職が人狼（黒）かそれ以外（白）かを知る [cite: 36]。
* [cite_start]**しずか（騎士）**: 対象を夜の襲撃から守る。連続守護と自己守護は不可 [cite: 39]。
* [cite_start]**ドラミ（霊媒師）**: 気絶者の正確な役職を知る [cite: 42]。
* [cite_start]**ジャイ子（人狼キラー）**: 自ら動くことはないが、人狼に襲撃されると返り討ちにして人狼を気絶させる [cite: 45]。
* [cite_start]**セワシ**: 対象が前の夜に「誰に能力を使ったか」を知る（役職は不明） [cite: 48, 62]。
* [cite_start]**先生**: 対象のその日の行動を完全に足止めし、無効化する（連続・自己指定不可） [cite: 64]。
* [cite_start]**のび太**: UI上は村人の誰かの役職が表示され能力も使えるが、結果はすべてデタラメである。ただし、自身を「しずか」だと思いこんでいる場合のみ、「ゴルゴンの首」の襲撃を防ぐことができる [cite: 67, 69, 76]。

### 🎭 第三者陣営

[cite_start]それぞれ固有の勝利条件を満たした時点で単独勝利となる [cite: 20]。

* [cite_start]**出木杉**: 対象の役職を推理する。的中すれば対象を気絶させ、外れれば自分が気絶する [cite: 79]。
* [cite_start]**ゴルゴンの首**: 対象を石化（気絶）させる。「しずか」の守護を貫通するが、「偽しずか（のび太）」の守護にだけ阻まれる [cite: 82]。
* [cite_start]**コピーロボット**: 対象を気絶させ、その役職と勝利条件を完全にコピーする。のび太をコピーした場合は、勘違い役職ごと引き継ぐ [cite: 85, 86]。
* [cite_start]**はる夫**: 誰か1人を「安雄」に指定する。自分の生死にかかわらず、「安雄」がゲーム終了時まで生存していれば勝利 [cite: 88, 89]。
* [cite_start]**神成さん**: 行動ターンはない。生存している間、毎ターン自分以外の誰か1人をランダムで気絶させる（先生の足止めも効かない）。この役職が場にいると、昼のターンに噂が流れる [cite: 91]。

## 技術スタック

* HTML5 / CSS3 (Responsive Design)
* JavaScript (ES6 Modules)
    * `gameState.js`: 状態管理
    * `roles.js`: 役職ロジック・優先度制御
    * `main.js`: ゲーム進行・判定コントローラー
    * `uiManager.js`: DOM操作・画面描画

## 開発・改修について

コードをローカルで動かす場合は、JavaScriptのモジュール（ES6）を使用しているため、ローカルサーバー（VSCodeのLive Server等）経由で `index.html` を開く必要がある。
